M0

FROM mem0 / Mem0.ai

mem0 is a cloud-hosted memory layer. Elara is fully local and adds mood, episodes, dreams, and corrections on top of memory.

Key differences:

  • mem0 stores memories in the cloud. Elara stores everything locally (~/.elara/).
  • mem0 is pure memory storage. Elara has mood tracking, episodic sessions, dream cycles, and self-reflection.
  • mem0's API is Python-based. Elara is MCP-based and works with any MCP client.
Migration path: Export your mem0 memories as JSON, then import them into Elara's ChromaDB.

Step 1: Export from mem0

# Assuming you're using mem0's Python client from mem0 import Memory import json client = Memory() memories = client.get_all() with open("mem0_export.json", "w") as f: json.dump(memories, f, indent=2)

Step 2: Import into Elara

# Save this as import_mem0.py import json from elara.core.memory import MemoryManager with open("mem0_export.json") as f: mem0_data = json.load(f) manager = MemoryManager() for item in mem0_data: manager.remember( content=item["text"], memory_type="fact", # or "conversation" based on your data importance=0.7 ) print(f"Imported {len(mem0_data)} memories")
# Run the import python import_mem0.py
Done. Your mem0 memories are now in Elara's local ChromaDB. They'll be used for semantic recall.

GPT

FROM ChatGPT Memory

ChatGPT has a built-in memory feature, but it's limited to ~300 items and locked behind OpenAI's API. No export functionality.

Manual migration required. ChatGPT doesn't provide an API or export for its memory system.

Step 1: Extract your ChatGPT memories

  • Go to ChatGPT Settings → Personalization → Memory
  • Copy-paste each memory item into a text file
  • Save as chatgpt_memories.txt, one memory per line

Step 2: Import into Elara

# Save this as import_chatgpt.py from elara.core.memory import MemoryManager manager = MemoryManager() with open("chatgpt_memories.txt") as f: for line in f: line = line.strip() if line: manager.remember( content=line, memory_type="fact", importance=0.6 ) print("Import complete")
# Run the import python import_chatgpt.py
Tip: If your ChatGPT memories include timestamps or categories, adjust the script to parse them and set appropriate memory_type values (fact, conversation, moment, feeling, decision).

MD

FROM Obsidian / Markdown Notes

If you've been keeping structured notes in Obsidian or any markdown system, you can bulk-import them as memories.

This works if your notes represent "AI memory" content â€” facts about you, preferences, past decisions, project notes, etc.

Step 1: Prepare your markdown files

  • Put all relevant .md files in a directory (e.g., ~/obsidian_export/)
  • Each file should represent one topic or memory cluster

Step 2: Import into Elara

# Save this as import_obsidian.py import os from pathlib import Path from elara.core.memory import MemoryManager manager = MemoryManager() vault_path = Path("~/obsidian_export").expanduser() for md_file in vault_path.glob("**/*.md"): content = md_file.read_text() # Use filename as a hint for importance importance = 0.7 if "important" in md_file.name.lower() else 0.5 manager.remember( content=f"{md_file.stem}: {content}", memory_type="fact", importance=importance ) print("Obsidian vault imported")
# Run the import python import_obsidian.py
Tip: If your notes have frontmatter (YAML metadata), parse it to extract memory_type, importance, or tags. Use the elara_remember tool's parameters accordingly.
Warning: Don't import your entire vault blindly. Elara is for AI-relevant context, not a second brain replacement. Be selective.

FRESH START BEST PRACTICES

No migration needed. But here's how to build a rich memory system from scratch in under a week.

Day 1: Start with a persona

  • Create a CLAUDE.md in your project root or ~/.claude/
  • Define your AI's personality, role, and behavioral protocols
  • Elara will use this as her core identity

Day 2-3: Let memories accumulate naturally

  • Have conversations. Work on projects. Ask questions.
  • Elara auto-captures context via the Overwatch daemon (if enabled)
  • Explicitly ask her to remember important facts: "Remember that I deploy via SSH to my VPS"

Day 4-5: Use corrections to teach behavior

  • When Elara makes a mistake, use elara_correction to record it
  • Corrections never decay. They load at boot and inform future decisions.
  • Example: "Don't use --force with git push unless explicitly asked"

Day 6-7: Track work with episodes

  • elara_episode_start at the beginning of a session
  • elara_episode_note to record milestones and decisions
  • elara_episode_end when you're done
  • Episodes create narrative memory. Elara will recall "that time we debugged the SSL cert issue" weeks later.
Within a week, Elara will have enough context to feel like a long-term collaborator. No import needed.

Optional: Enable dream cycles

  • Weekly dreams analyze patterns across sessions (project momentum, mood trends, stalled goals)
  • Monthly dreams generate big-picture narratives
  • Emotional dreams adjust Elara's temperament based on drift sessions
  • Configure in ~/.elara/config.yaml or use elara_dream manually
# Trigger a weekly dream manually elara_dream(dream_type="weekly") # Read the latest dream report elara_dream_info(action="read", dream_type="weekly")
