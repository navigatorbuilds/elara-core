// PYTHON API

Programmatic access to 6 core modules. Use the underlying Python classes directly — no MCP required.

API reference is derived from source and may not reflect exact signatures in every release. Always check the source code. Elara Core's internal module layout is daemon.* and memory.* — not a flat namespace.

MEM

VectorMemory

memory/vector.py

Semantic memory store backed by ChromaDB. Memories are vectorized for meaning-based recall, and tagged with emotional context at encoding time. Retrieval supports mood-congruent boosting — memories encoded during similar emotional states surface first.

from memory.vector import VectorMemory, remember, recall

// Class methods

remember(content: str, memory_type: str = "conversation", importance: float = 0.5, metadata: Optional[Dict] = None, tag_with_emotion: bool = True)
Store a memory. Content is vectorized and indexed in ChromaDB. If tag_with_emotion is True, the current emotional state (valence, energy, openness, discrete emotion label) is captured alongside the memory.
Returns: str — Memory ID
memory_type: "conversation" | "fact" | "moment" | "feeling" | "decision" importance: 0.0 – 1.0
recall(query: str, n_results: int = 5, memory_type: Optional[str] = None, min_importance: float = 0, mood_weight: float = 0.3)
Search memories by semantic similarity with mood-congruent boosting. mood_weight controls ranking bias: 0 = pure semantic, 1 = heavily mood-biased. Internally fetches 3x results and re-ranks using emotional resonance scores.
Returns: List[Dict] — Memories with relevance, resonance, and combined_score fields
recall_by_feeling(feeling: str, n_results: int = 5)
Recall memories matching a feeling or emotional tone. Uses heavy mood weighting (0.6).
Returns: List[Dict]
recall_mood_congruent(n_results: int = 5)
Get memories that match current mood without a specific query. Used for spontaneous recall, nostalgia, or rumination. Ranks purely by emotional resonance.
Returns: List[Dict]
recall_recent(days: int = 7, n_results: int = 10)
Get recent memories regardless of query, sorted by timestamp.
Returns: List[Dict]
forget(memory_id: str)
Remove a specific memory by ID.
Returns: bool
count()
Total number of stored memories.
Returns: int

MOD

Mood Functions

daemon/mood.py

Three-dimensional emotional state engine: valence (-1 to 1), energy (0 to 1), openness (0 to 1). Mood decays naturally toward a baseline temperament over time. Supports discrete emotion labels via a circumplex-style vocabulary (36 emotions). Not a class — uses module-level functions operating on a shared JSON state file.

from daemon.mood import get_mood, adjust_mood, create_imprint, describe_mood

// Core functions

get_mood()
Get current mood state with time decay applied. Returns the raw numerical values.
Returns: dict — {"valence": float, "energy": float, "openness": float}
adjust_mood(valence_delta: float = 0, energy_delta: float = 0, openness_delta: float = 0, reason: Optional[str] = None, create_imprint: bool = False, imprint_strength: float = 0.5)
Shift mood by deltas. Use small values (0.05 – 0.2). Optionally create an emotional imprint at the same time. The reason is logged to the mood journal and stored as emotional residue.
Returns: dict — Updated mood state
set_mood(valence: Optional[float] = None, energy: Optional[float] = None, openness: Optional[float] = None, reason: Optional[str] = None)
Set mood to absolute values. None values keep current. Used by personality mode switching.
Returns: dict — Updated mood state
create_imprint(feeling: str, strength: float = 0.7, imprint_type: str = "moment")
Create an emotional imprint — a persistent feeling that outlasts session details. Imprints decay slowly and color future interactions. Up to 20 are retained.
Returns: dict — The imprint record
describe_mood()
Human-readable mood description using the emotion vocabulary. Returns natural language like "Feeling warm and present. Calm energy, very open."
Returns: str
get_current_emotions()
Full emotion readout: primary emotion, secondary, blend, quadrant, and any imprints being carried.
Returns: dict — {"primary", "secondary", "blend", "quadrant", "emotions", "raw", ...}
get_session_arc()
Analyze the emotional arc of the current session. Classifies the trajectory as upswing, slow_drain, recovery, crash, rollercoaster, steady, or flat.
Returns: dict — {"pattern", "description", "start_emotion", "end_emotion", "valence_delta", ...}
get_imprints(min_strength: float = 0.2)
Get active emotional imprints above a strength threshold.
Returns: List[dict]
get_full_state()
Complete emotional state including mood, imprints, residue, flags, consolidation, and allostatic load.
Returns: dict
EXAMPLE
from daemon.mood import get_mood, adjust_mood, create_imprint, describe_mood # Check current mood mood = get_mood() print(f"Valence: {mood['valence']}, Energy: {mood['energy']}") # Shift mood after good progress adjust_mood(valence_delta=0.1, energy_delta=0.05, reason="shipped the feature") # Human-readable description print(describe_mood()) # "Feeling content and warm. Steady energy, open." # Create a lasting imprint create_imprint("meaningful late-night debugging session", strength=0.8) # Full emotion readout from daemon.mood import get_current_emotions emo = get_current_emotions() print(f"{emo['primary']} ({emo['blend']}), quadrant: {emo['quadrant']}")
EPI

EpisodicMemory

memory/episodic/

Rich autobiographical memory for work sessions. Stores episodes with milestones, decisions, mood trajectory, project context, and auto-generated narrative summaries. Built from five mixins: Core, Lifecycle, Retrieval, Threading, and Compression. Episodes are stored as individual JSON files, indexed by date and project, with milestones in ChromaDB for semantic search.

from memory.episodic import EpisodicMemory, get_episodic, milestone, decision

// Lifecycle methods

create_episode(episode_id: str, session_type: str, started: str, projects: List[str] = None, mood_at_start: dict = None, continues_from: str = None)
Create a new episode record. Automatically chains to previous episode and indexes by date and project.
Returns: dict — The full episode record
session_type: "work" | "drift" | "mixed"
add_milestone(episode_id: str, event: str, milestone_type: str = "event", importance: float = 0.5, metadata: dict = None)
Add a milestone to an episode. Automatically tagged with current mood and indexed in ChromaDB for semantic search.
Returns: dict — The milestone record
milestone_type: "event" | "decision" | "milestone" | "insight" | "error"
add_decision(episode_id: str, what: str, why: str = None, alternatives: List[str] = None, confidence: str = "medium", project: str = None)
Record a decision made during the episode. Also creates a high-importance milestone automatically.
Returns: dict — The decision record
confidence: "low" | "medium" | "high"
close_episode(episode_id: str, summary: str = None, narrative: str = None, mood_end: dict = None)
Close an episode, finalizing its record. Calculates duration, mood delta, and auto-generates a narrative summary (tries local LLM first, falls back to template).
Returns: dict — The finalized episode
get_episode(episode_id: str)
Get a specific episode by ID.
Returns: Optional[dict]
get_stats()
Episodic memory statistics: total episodes, projects tracked, milestone count.
Returns: dict

// Convenience functions (module-level)

milestone(event: str, **kwargs)
Quick milestone recording for the current active episode. Uses the global singleton.
Returns: dict
decision(what: str, **kwargs)
Quick decision recording for the current active episode.
Returns: dict
EXAMPLE
from memory.episodic import EpisodicMemory, milestone, decision from datetime import datetime # Create instance ep = EpisodicMemory() # Start an episode episode_id = datetime.now().strftime("%Y-%m-%d-%H%M") ep.create_episode( episode_id=episode_id, session_type="work", started=datetime.now().isoformat(), projects=["elara-core"], ) # Record milestones ep.add_milestone(episode_id, "Completed API docs page", importance=0.7) # Record a decision ep.add_decision( episode_id, what="Use module-level functions instead of class for mood", why="Simpler API, state is file-backed anyway", confidence="high", ) # Close the episode result = ep.close_episode(episode_id, summary="Built API docs") print(f"Duration: {result['duration_minutes']}min, mood delta: {result['mood_delta']}") # Quick convenience (auto-uses current episode) milestone("Fixed the flaky test", importance=0.6)
COR

Corrections

daemon/corrections.py

Learn from mistakes. Corrections never decay — they're permanent records of behavioral tendencies and technical patterns that should not be repeated. Backed by JSON (source of truth) with ChromaDB semantic indexing for context-aware matching. Before doing similar work, check_corrections() finds relevant warnings.

from daemon.corrections import add_correction, check_corrections, list_corrections

// Public functions

add_correction(mistake: str, correction: str, context: Optional[str] = None, correction_type: str = "tendency", fails_when: Optional[str] = None, fine_when: Optional[str] = None)
Record a correction. fails_when and fine_when prevent overgeneralization — the same pattern might be correct in different contexts.
Returns: dict — The correction entry
correction_type: "tendency" (behavioral) | "technical" (code pattern)
check_corrections(task_description: str, n_results: int = 3)
Semantic search: find corrections relevant to what you're about to do. This is the activation function — it matches corrections to your current task context, not just keywords. Fails loud if ChromaDB is down (returns error info rather than silently skipping).
Returns: List[dict] — Matching corrections with relevance score and fails_when/fine_when conditions
list_corrections(n: int = 20)
Get the most recent corrections.
Returns: List[dict]
record_activation(correction_id: int, was_relevant: bool = True)
Track that a correction was surfaced during work. Updates last_activated, times_surfaced, and times_dismissed counters.
get_dormant_corrections(days: int = 14)
Find corrections that have never been activated or haven't fired in N days. Used by the blind_spots analysis.
Returns: List[dict]
boot_corrections(n: int = 10)
Get corrections formatted for boot loading. Tendencies always show; technical corrections only if recently activated (last 7 days).
Returns: str — Formatted list
ensure_index()
Ensure ChromaDB index is in sync with the JSON file. Call at boot.
Returns: str — Status message
EXAMPLE
from daemon.corrections import add_correction, check_corrections # Record a mistake add_correction( mistake="Used .toList() on Firestore query", correction="Use .get() instead for single reads", context="HandyBill invoice loading", correction_type="technical", fails_when="reading a single document by ID", fine_when="actually need a list of documents from a collection query", ) # Check before doing similar work warnings = check_corrections("Writing a Firestore query to load invoices") for w in warnings: print(f" {w['mistake']} -> {w['correction']} (relevance: {w['relevance']:.2f})") if w.get("fails_when"): print(f" Fails when: {w['fails_when']}")
GOL

Goals

daemon/goals.py

Persistent goal tracking across sessions. Simple JSON storage, no database. Goals have status (active, stalled, done, dropped), priority (high, medium, low), and optional project association. Checked at boot, with stale detection for goals untouched in 7+ days.

from daemon.goals import add_goal, list_goals, update_goal, boot_summary

// Public functions

add_goal(title: str, project: Optional[str] = None, notes: Optional[str] = None, priority: str = "medium")
Add a new goal. Automatically assigned an incrementing ID and set to "active" status.
Returns: dict — The goal record
priority: "high" | "medium" | "low"
update_goal(goal_id: int, status: Optional[str] = None, notes: Optional[str] = None, priority: Optional[str] = None, title: Optional[str] = None)
Update a goal's status, notes, priority, or title. Automatically touches the last_touched timestamp.
Returns: dict — Updated goal or {"error": "..."}
status: "active" | "stalled" | "done" | "dropped"
list_goals(status: Optional[str] = None, project: Optional[str] = None)
List goals, optionally filtered by status and/or project.
Returns: List[dict]
get_goal(goal_id: int)
Get a single goal by ID.
Returns: Optional[dict]
stale_goals(days: int = 7)
Find active goals not touched in N days.
Returns: List[dict]
touch_goal(goal_id: int)
Update a goal's last_touched timestamp without changing anything else.
boot_summary()
Quick summary for session boot. Shows active goals sorted by priority, stale goals, and recently completed goals.
Returns: str — Formatted summary
EXAMPLE

COG

Reasoning Trails

daemon/reasoning.py

Track hypothesis → evidence → conclusion → outcome chains. When debugging complex problems, record the reasoning trail so next time a similar problem appears, you can search past trails instead of re-discovering the same solution. Each trail is stored as an individual JSON file, with ChromaDB semantic indexing for problem-similarity search.

from daemon.reasoning import start_trail, add_hypothesis, update_hypothesis, solve_trail, search_trails

// Core functions

start_trail(context: str, tags: Optional[List[str]] = None)
Start a new reasoning trail for a problem under investigation. Generates a unique trail ID (SHA-256 hash) and indexes the context in ChromaDB.
Returns: dict — The trail record with trail_id
add_hypothesis(trail_id: str, hypothesis: str, evidence: Optional[List[str]] = None, confidence: float = 0.5)
Add a hypothesis to an existing trail with optional initial evidence and confidence level.
Returns: dict — Updated trail
update_hypothesis(trail_id: str, hypothesis_index: int, outcome: Optional[str] = None, evidence: Optional[List[str]] = None, confidence: Optional[float] = None)
Update a hypothesis's outcome, add new evidence, or adjust confidence.
Returns: dict — Updated trail
outcome: "true" | "false" | "partial" hypothesis_index: 0-indexed
abandon_approach(trail_id: str, approach: str)
Record an approach that was tried and dropped. Tracked for analytics (abandonment rate).
Returns: dict — Updated trail
solve_trail(trail_id: str, solution: str, breakthrough_trigger: Optional[str] = None, tags: Optional[List[str]] = None)
Mark a trail as solved. Records the final solution and optionally what triggered the breakthrough. Tags are merged with existing tags.
Returns: dict — The solved trail
search_trails(query: str, n: int = 5)
Search past reasoning trails by problem similarity using ChromaDB semantic search. Falls back to keyword search if ChromaDB is unavailable.
Returns: List[dict] — Trails with _similarity score
list_trails(resolved: Optional[bool] = None, tag: Optional[str] = None, n: int = 20)
List trails, optionally filtered by resolved status or tag. Sorted by most recent first.
Returns: List[dict]
get_active_trail()
Get the most recent unresolved trail (if any).
Returns: Optional[dict]
get_recurring_problem_tags(min_count: int = 3)
Find tags appearing in multiple trails — recurring problem areas. Used by blind_spots analysis.
Returns: List[dict] — {"tag", "count", "trail_ids"}
get_abandonment_rate()
Stats on how often approaches are abandoned across trails.
Returns: dict — {"total_trails", "abandoned_total", "avg_per_trail", "unresolved"}
EXAMPLE
from daemon.reasoning import ( start_trail, add_hypothesis, update_hypothesis, abandon_approach, solve_trail, search_trails ) # Start investigating a problem trail = start_trail( context="Memory search returning empty results after reboot", tags=["chromadb", "memory"], ) tid = trail["trail_id"] # Form hypotheses add_hypothesis(tid, "ChromaDB collection might be corrupted", confidence=0.6) add_hypothesis(tid, "Data dir path changed between sessions", confidence=0.3) # Add evidence update_hypothesis(tid, hypothesis_index=0, evidence=["rebuild_indexes fixed it", "collection.count() was 0 before rebuild"], confidence=0.9, outcome="true") # Record dead end abandon_approach(tid, "Tried deleting and recreating the collection manually") # Solved solve_trail(tid, solution="ChromaDB index was stale after state file migration", breakthrough_trigger="rebuild_indexes command", ) # Next time a similar problem appears... past = search_trails("memory recall returns nothing") for t in past: print(f" [{t['_similarity']:.2f}] {t['context'][:60]}") if t.get("final_solution"): print(f" Solution: {t['final_solution']}")
