[ WHAT WE COLLECT ]

Nothing.

We don't collect usage data. We don't log errors remotely. We don't track installations. We don't know who you are, what you're working on, or how you're using Elara.

There are no analytics packages. No phone-home mechanisms. No "anonymous" statistics. The code is open source — verify it yourself.

[ WHERE YOUR DATA LIVES ]

All data stays on your machine, in your local filesystem. Default location: ~/.elara/

You control this directory. You can back it up, delete it, encrypt it, move it to a USB drive — whatever you want. It's yours.

What's stored locally:

  • Semantic memories — ChromaDB vector indexes (embedded, no external database)
  • Conversation history — JSONL text files
  • Episode logs — Session data in JSON format
  • Mood state — Persona file tracking emotional state
  • Corrections — Mistakes you've taught Elara to avoid
  • Goals — Active, stalled, and completed goals
  • Reasoning trails — Problem-solving chains
  • Dream reports — Pattern analysis summaries
  • Gmail cache — If you enable Gmail integration, email data cached locally
  • RSS feed cache — If you add feeds, fetched items stored locally

ChromaDB runs in embedded mode. There's no external database server, no network connections, no cloud sync. Just local SQLite + vector index files.

[ OPTIONAL INTEGRATIONS ]

Elara includes optional features that you must explicitly enable. Here's what they do:

  • Gmail Integration — Uses YOUR Google OAuth2 credentials. The token is stored locally in your data directory. Emails are fetched via Google's API and cached on your machine. We don't see your credentials or your emails.
  • RSS Feeds — You provide URLs. Elara fetches them directly from the source and stores items locally. No middleman, no aggregator, no tracking.
  • Local LLM (Ollama) — If you install Ollama separately, Elara can call it for lightweight tasks. All inference happens locally on your machine.
You control these features. They're off by default. If you never enable them, they don't exist.
[ NO CLOUD SERVICES ]

Elara doesn't use any cloud backend. There's no SaaS component, no API server we operate, no hosted database.

It's a Python package that runs on your machine via the MCP protocol. The only network activity is what you configure (Gmail, RSS feeds, etc.).

We don't have servers. We couldn't collect your data even if we wanted to — there's nowhere for it to go.

[ WHAT WE DON'T DO ]
  • No usage analytics — We don't track how often you use it, which tools you call, or what you're working on.
  • No error reporting — If something crashes, it stays on your machine. Fix it or file a GitHub issue.
  • No anonymous data — There's no "opt-out" telemetry toggle because there's nothing to opt out of.
  • No IP logging — We don't run a server. Your IP never hits our infrastructure.
  • No accounts — No sign-ups, no logins, no emails, no passwords.
  • No cloud sync — If you want sync, set up your own solution (rsync, Syncthing, Dropbox, whatever).
Important: If you enable Gmail integration, your OAuth token is stored locally in plaintext. Protect your data directory. If someone gets access to your machine, they can read your cached emails.
